5. Angular Templates

Created Saturday 10 July 2021

Template is the html file tagged to any particular component. It describes the basic structure of the component and how it will be shown in the browser.

Templates are basically static HTML combined with some dynamic components that are provided by Angular in the form of logic.

Passing Data to Components via the Template

To go from static to dynamic components, the first thing to do is to pass some dynamic data to the component. This is done through:

  1. Interpolation
Add an attribute inside component class and use it.

Refer to it inside template using 

{{ attr }}

We can optionally use pipes in template like 

{{ attr | uppercase }}

  1. Data Binding
    1. Property Binding: Allows us to dynamically change the HTML properties based on logic.

<button id = "newbutton">Click me!</button>

Just creates a basic HTML button. Now a button has certain properties. Let's check 'disabled' 

<button id = "newbutton" disabled>Click me!</button>

Now, dynamically: 

In TS:
isButtonDisabled : boolean = false;

In template:
<button id = "newbutton" 
[disabled]="isButtonDisabled">Click me!</button>

We have just "bound" the value of a variable to a property of an element in the DOM. This kind of data binding is one-way, source to view property binding. So updates in TS will reflect in the view but not vice versa.

  1. Event Binding: If we wanted the reverse, ie one-way, view to source event binding, that can be done using () brackets.

In TS:
buttonClicked(){
    console.log("Button has been clicked.")
  }

In template:
<button [disabled]='isButtonDisabled' 
(click) = 'buttonClicked()'>Click me!</button>

This allows you to bind the event of a click with an action in the TS. 

In template:
<input type='text' name='text' (input) = "formFilled($event)">

In TS:
formFilled(event: any){
    console.log(event); 
    this.textEntered= (<HTMLInputElement>event.target).value;
  }

This is also a one-way data-binding where data is sent from view to source.

  1. Two-Way Data Binding
Changes in data made in the view are updated in the model/logic, and vice versa.

Let's say I want to take the username as input, but if I change it in the logic that should be the name as well.

For this, we import 

import { FormsModule } from '@angular/forms';

in component ts as well as in appmodulets. Additionally, must add FormsModule to imports array of appmodule. 

imports: [
    BrowserModule,
    FormsModule
  ]

Now, 

In TS:
textEntered : any = '';

In template:
<input type='text' name='text' [(ngModel)] = "textEntered">
<br><br>
<div>
    {{ textEntered }}
</div>

This allows for live update. The difference between one-way event driven data binding and two-way binding is that updates on the model from source are not reflected on the view end for it.

If we had two inputs, one not using 2WDB and one using it, updating the first input would update the output of the second but not vice versa

Write but not read = one way event driven binding

Read but not write = string interpolation/one way data driven binding

Write and read = two way data binding
