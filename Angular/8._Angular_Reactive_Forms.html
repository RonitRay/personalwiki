8. Angular Reactive Forms

Created Saturday 10 July 2021

When we use ngModel with an input field, we are already using an Angular form! These are called Angular template-driven forms and are the most basic way of dealing with form data on Angular.

However, template-driven forms don't scale well and don't utilize the full power of Angular, instead falling back on the HTML + JS approach to things.

In Angular, we can use the more robust Angular Reactive Forms feature.

Advantages:

  1. Single configuration point for entire form
  2. Much better and more versatile validation options
  3. Access to useful features like checking if entire form (all fields) is valid

Before we start:

Add the ReactiveFormsModule to your imports array in app.module.ts

Check that it has also been automatically imported at the top of this file or it will throw error.

In your component TS file, import this: 

import { FormGroup } from '@angular/forms';

Let us now create a form using Reactive Forms. The crux of this is that

  1. The form is first defined as a FormGroup object in TypeScript, then actually created and bound to it in the template.
  2. Each field is given a FormControl, that has an initial value and some validations attached to it.

export class FormComponent implements OnInit {

  loginForm: FormGroup;
.
.
.
constructor() {
    this.loginForm = new FormGroup({
      'email' : new FormControl(null, [Validators.required, Validators.email]),      
      'password' : new FormControl(null, Validators.required)
    });
  }

Now we add the bits to the template to make it reactive.

  1. The form is given the formGroup as a property binding
  2. Every individual form field is given a form control name as a property binding or as a simple string (both are valid)

<section class='container'>
    <div class='regForm'>
        <h3>User Registration</h3>
        <form class="registration-form" [formGroup]="loginForm">
            <mat-form-field appearance="fill">
                <mat-label>Username</mat-label>
                <input matInput placeholder="Ex. ronray" formControlName='username'>
            </mat-form-field>
            <br>
            <mat-form-field appearance="fill">
                <mat-label>Password</mat-label>
                <input matInput formControlName='password'>
            </mat-form-field>
        </form>
        <div class='action'>
            <button mat-raised-button color="primary" [disabled]='!loginForm.valid' (click)="onSubmit()">Register</button>
        </div>
        <br><br>
        <div class='data' *ngIf="areDetailsShown">
            <h4>Submitted Data</h4>
            <p>Username: {{ loginForm.value.username }}</p>
            <p>Password: {{ loginForm.value.password  }}</p>
        </div>
    </div>
</section>


import { Component, OnInit } from '@angular/core';
import { FormControl, FormGroup, Validators } from '@angular/forms';

@Component({
  selector: 'app-form',
  templateUrl: './form.component.html',
  styleUrls: ['./form.component.css']
})
export class FormComponent implements OnInit {

  loginForm: FormGroup;
  areDetailsShown: boolean;

  constructor() {
    this.loginForm = new FormGroup({
      'email' : new FormControl(null, [Validators.required, Validators.email]),      
      'password' : new FormControl(null, Validators.required)
    });
    this.areDetailsShown = false;
  }

  ngOnInit(): void {

  }

  onSubmit() : void {
    this.areDetailsShown = true;
    console.log(this.loginForm.value);
  }
}

The extra logic to show details on submission is just added to show that our form is working and we can retrieve data from it.
