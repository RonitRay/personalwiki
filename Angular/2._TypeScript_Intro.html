2. TypeScript Intro

Created Saturday 10 July 2021

TypeScript is an extension of JavaScript.

[Typed JavaScript at Any Scale.](https://www.typescriptlang.org/)

  1. It allows more robust development at scale
  2. It can be considered a "superset" meaning it has all the features of JS, but adds some killer features to it.
    1. Static Typing
    2. Strict Checks

function add(a,b) {
	var c= a + b
	return c;
}

console.log(add(1,2));
console.log(add(1,'2'));

JS is not untyped, it is dynamically typed: no types expected, but type is determined at runtime.

Issues:

- Will accept and try to compute any input

- If unexpected or edge cases are not handled properly, there is scope for serious breakage
- JS code is easy to write, but not easy for another person to understand. 

function addTwoNumbers (num1: number, num2: number) : number {
    let sum : number = num1 + num2;
    return sum;
}

console.log(addTwoNumbers(1,2));
console.log(addTwoNumbers(1,'2'));

let arrayOfStrings : string[];

arrayOfStrings : ['a','b','c'];

let employee : {
	name: string;
	employeeId: number;
};

employee = {
	name: 'Ron',
	employeeId: '123'

Generics and Other Power Features 

function insertAtBeginning(array: string[], value:string) {
	const newArray = [value, ... array];
	return newArray;
}

const demoArray = [1,2,3];
const updatedArray = insertAtBeginning(demoArray, -1);

updatedArray[0].split(' ');

The above is valid TypeScript, we define a func to add an element to beginning of an array, no matter what the type of array is.

However, if we then want to do type-specific operations with the array elements, we lose out on TS checking due to the explicit any assignment.

Even if our array has numbers, the line where we call split (valid only on Strings) is not flagged by TSC because at compile time TSC does not know what the type of the element will be. We get a runtime error instead, which is exactly what TS is expected to avoid.

To deal with this, we have the concept of generics. 

function insertAtBeginning<T>(array: T[], value:T) {
	const newArray = [value, ... array];
	return newArray;
}

const demoArray = [1,2,3];
const addedElement = -1;
const updatedArray = insertAtBeginning(demoArray, addedElement);

updatedArray[0].split('');

There are other features TS uses that are in line with languages like Java like:

  1. Proper classes with typed attributes, constructors etc.
  2. Interfaces to define custom types that can then be implemented by classes to enforce a consistent structure.
  3. Access and scope control (private etc.)

Browsers do not run TypeScript by default. As a result, we need to change all our TS code to JS before running it. TS provides a "transpilation" mechanism using the TypeScript compiler tsc to do this.

The TypeScript compiler will always flag issues at compile-time, but to make the development process even more convenient, we can use a code editor that compiles everything as you type, flagging any errors the moment they come up.

NOTE: The strict checks that TS enforces are not part of the JavaScript output. This is why you write in safe TS to ensure any mistakes are handled well before the output JS is generated.
