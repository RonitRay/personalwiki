6. Angular Directives

Created Saturday 10 July 2021

Angular Directives allow us to add some logic or behaviour to the templates and control how elements are shown and rendered.

We have already used one directive in the last lesson, the ngModel directive allows for 2 way data binding.

Attribute Directives

  1. NgModel: already discussed
  2. NgClass: allows you to dynamically add CSS classes to any DOM element.

<div [ngClass]="currentClasses">
Some content here
</div>

currentClasses: Record<string, boolean> = {};
/* . . . */
  setCurrentClasses() {
    // CSS classes: added/removed per current state of component properties
    this.currentClasses =  {
      saveable: this.canSave,
      modified: !this.isUnchanged,
      special:  this.isSpecial
    };
  }

  1. NgStyle: allows you to dynamically add CSS styles to any DOM element.

<div [ngStyle]="currentStyles">
  Some content
</div>

currentStyles: Record<string, string> = {};
/* . . . */
  setCurrentStyles() {
    // CSS styles: set per current state of component properties
    this.currentStyles = {
      'font-style':  this.canSave      ? 'italic' : 'normal',
      'font-weight': !this.isUnchanged ? 'bold'   : 'normal',
      'font-size':   this.isSpecial    ? '24px'   : '12px'
    };
  }

Structural Directives (prepend with *)

  1. If directive: If true, element is shown. If false, not shown.

<button *ngIf='isButtonShown' 
[disabled]='isButtonDisabled' 
(click) = 'buttonClicked()'>Click me!</button>

  1. For directive: Element is repeated as many times as the loop runs/the satisfying condition is true.

  1. There is also ngSwitch directive to pick an item from a list.
