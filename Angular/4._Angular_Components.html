4. Angular Components

Created Saturday 10 July 2021

Angular is highly modular and encourages you to think of your app as a loosely-linked collection of different independent components.

In the design phase, you are expected to decompose your app into atomic components, and then define a logical flow that ties them together.

The only page loaded by the angular application is index.html.

Along with this, some JS is injected by default. This JS is what manipulates the content of the page based on logic you have defined already. 

<app-root></app-root>

App root is a custom HTML tag, obviously. This refers to a very particular component in the project. At the time of creating a new project, only a single component is created called the AppComponent.

Each Angular component has:

- An HTML template that declares what renders on the page
- A Typescript class that defines behavior
- A CSS selector that defines how the component is used in a template
- Optionally, CSS styles applied to the template

What does ng serve do?

When ng serve is called, it:

  1. Launches a live server on [localhost](http://localhost) to show changes automatically as they happen.
  2. Creates "script bundles" of JavaScript that are injected into your basic HTML file
  3. All execution starts at main.ts

main.ts description: 

platformBrowserDynamic().bootstrapModule(AppModule)

bootstrapModule basically allows us to inject or instantiate something called AppModule within our application.

AppModule refers to app.module.ts

Here the bootstrap attribute will consist of all the elements that your application will be expected to know about when showing the index.html file.

Top-Down execution:

index.html

main.ts > bootstrapModule > AppModule.bootstrap > AppComponent > find template, manipulate template, render output in index.

The root component or AppComponent is the base of the entire application and will encapsulate any other components we make.

Only AppComponent is typically bootstrapped on application start and included in index.html. Everything else is added to app.component.html and lower level components.

To create a new component, we type: 

ng generate component <component-name>

OR

ng g c <component-name>

And then we can refer to it in any other component using the defined selector.
