1. What Is Angular

Created Saturday 10 July 2021

Angular is a development platform/library used for front-end (client-side) web development. It is an in-house tool developed by Google and allows us to create reactive, single-page-applications (SPA).

Multi-page App:

[Home | Ronit Ray](http://ronitray.xyz)

  1. Standard for the web for a long time.
  2. Every time we need a new page, browser sends request to server, server sends response with necessary content.
  3. Page reloads every time, and control goes from page > browser > page and so on.
  4. Not a smooth, consistent interface, but sometimes faster.

Single-page App:

[YouTube](https://youtube.com)

  1. On first load, the browser downloads an "application", typically in JS, instead of just the index/landing page.
  2. Most of the computation that happens is now done by the application client/server side, and appropriate changes are rendered within the app without ceding control to the browser.
  3. Load, don't reload
  4. Smooth, consistent UI.

SPA frameworks:

  1. Angular
  2. React
  3. Vue.js
  4. Svelte
  5. Ember.js

and many, many more

Why Angular?

  1. Well-established
  2. Backed by major corporation
  3. Used by huge network of developers
  4. Developer-friendly
  5. Can be extended into native apps using hybrid frameworks

2010: AngularJS initial launch

AngularJS was a JavaScript-based library that was added to your projects the same way you add jQuery/Bootstrap, as in using 

<script src="...">

2016: Angular 2 initial release

Angular 2 is a ground-up rewrite of Angular using TypeScript instead of JavaScript, designed to fix a number of issues.

[TypeScript Intro](https://www.notion.so/TypeScript-Intro-f25d6425308a4bdd96aa82dfaa14ddda)

New versions are released every 6 months but there are no major jumps on the scale of 1>2.

NOTE: Whenever we refer to Angular from here on, we are only talking about Angular 2+.
